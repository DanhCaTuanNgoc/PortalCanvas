# PortalCanvas Phase 1: Foundation

## Goal

Set a clean, stable baseline for PortalCanvas so feature development can move quickly without repo drift or unclear structure.

## Scope

### In scope

- Root project documentation.
- Web app setup and run instructions.
- Repo hygiene: ignore rules, folder structure, and baseline scripts.
- Consistent naming for phase documents and implementation notes.

### Not in scope

- New canvas features.
- Contract node interactions.
- Runtime chain integration.
- Fee inspection logic.
- Data fetching or persistence layers.

## Foundation Tasks

1. Keep the root README focused on PortalCanvas.
2. Keep implementation and phase documents easy to navigate.
3. Make sure `apps/web` can be run and built from a clean checkout.
4. Ensure project-specific ignore rules exclude generated or unrelated artifacts.

## Current Status

- Root documentation updated.
- Phase 0 scope documented.
- Implementation roadmap present.
- Web app scaffold already in place and buildable.

## Phase 1 Completion Criteria

- Root README describes the product and local setup clearly.
- Phase docs exist and link to each other.
- The repo has a stable baseline for the next implementation phases.
